# 建筑能耗基因理论 —— CTM 特征温度法逐时冷热负荷计算程序

## 项目简介

本程序基于**建筑能耗基因理论**，采用 **CTM 特征温度法**（Characteristic Temperature Method），
实现建筑逐时空调冷负荷和供暖热负荷的计算。

程序采用 Python + Streamlit 开发，提供友好的网页交互界面。

## 理论依据

- 《建筑节能原理》—— 建筑能耗基因理论
- 式 6-26：基于关机总特征温度 t∞_shut 与设定温度的差值计算冷/热负荷
- CTM 特征温度法：将各分项得热折算为特征温度，合成总特征温度后代入公式

## 程序功能

1. 建筑参数输入与管理（默认值、JSON 导入导出）
2. 逐时气象数据输入（Excel 上传、CSV 上传、手动输入）
3. CTM 特征温度法逐时计算：
   - 传热反应系数计算
   - 太阳空气温度计算
   - 分项特征温度计算（7项）
   - 关机总特征温度合成
   - 式 6-26 显热冷负荷计算
   - 式 6-26 供暖负荷计算
   - 潜热附加项计算（与显热分开）
4. 结果输出：
   - 逐时结果表
   - 分项特征温度表
   - 日汇总、月汇总
   - 峰值小时核查
   - 中文图表（Plotly 交互式）
5. 导出功能：
   - 完整 Excel 文件（11个工作表）
   - 逐时结果 CSV
   - 日/月汇总 CSV
   - 参数 JSON
   - 输入模板 CSV

## 计算流程

```
输入建筑参数和逐时气象数据
    ↓
计算各分项传热反应系数 H（外墙、屋面、外窗、外门、通风）
    ↓
计算各朝向太阳空气温度 T_sa
    ↓
计算玻璃太阳得热 Q_glass 和内部得热 Q_internal
    ↓
计算 7 项分项特征温度 t_inf
    ↓
合成关机总特征温度 t∞_shut = Σt_inf
    ↓
代入式 6-26：冷负荷 = max(0, (t∞_shut - T_cool) × H_total)
代入式 6-26：热负荷 = max(0, (T_heat - t∞_shut) × H_total)
    ↓
额外计算潜热附加项（通风潜热 + 人员潜热）
    ↓
输出逐时结果、汇总、图表、Excel
```

## 输出结果说明

### 式 6-26 显热冷负荷（cooling_load_sensible_kw）

由关机总特征温度与空调设定温度的差值得到，为纯显热冷负荷。

### 供暖负荷（heating_load_kw）

由供暖设定温度与关机总特征温度的差值得到。

### 潜热附加项

- 通风潜热冷负荷（ventilation_latent_load_kw）
- 人员潜热冷负荷（people_latent_load_kw）

潜热附加项仅在显热冷负荷大于 0 时计入，不属于式 6-26 显热负荷。

### 空调总冷负荷（cooling_load_total_kw）

= 显热冷负荷 + 通风潜热 + 人员潜热

## 常见问题

### Q: 如何准备气象数据？

准备包含日期、时间、室外温度、相对湿度和各朝向太阳辐射的 CSV 或 Excel 文件。可参考 `sample_input.csv`。

### Q: 支持多少小时的数据？

支持 1 小时到 8760 小时（全年）的任意数据量。

### Q: 潜热为什么有时为 0？

当显热冷负荷为 0 时（即不需要空调制冷），潜热附加项也取 0。

### Q: 如何自定义建筑面积？

在「建筑参数」页面选择"手动输入"建筑面积模式。

## 注意事项

1. 本程序采用 CTM 特征温度法，非传统的得热叠加法
2. 冷负荷显热（式 6-26）与空调总冷负荷（含潜热）是不同指标
3. 默认时间步长为 1 小时，逐时 kW 数值上等于 kWh
4. 本程序为稳态简化模型，未考虑围护结构蓄热的动态响应
5. 太阳辐射数据应为逐时值，单位 W/m²
